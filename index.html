Two skills, one workflow.

Use image generation when the goal is stunning final pages. Use reconstruction when the goal is editable PowerPoint structure.

Easy To Image PPT

Generate final slide images from real source material.

Builds Chinese academic and report-style slide images through a gated workflow: source analysis, outline approval, style previews, Image2 generation, and QA.

  • Delivers ordered PNG slides, from slide_01.png onward.
  • Locks outline and visual direction before final generation.
  • Preserves strict figures: plots, screenshots, tables, and lab images.
Easy To Edit PPT

Reconstruct screenshots into editable PowerPoint decks.

Converts slide images into editable PPTX files by combining OCR, clean backgrounds, native object reconstruction, asset preservation, validation, and repair.

  • Creates editable text boxes and native shapes where reliable.
  • Keeps dense diagrams as movable assets when fidelity matters.
  • Returns the PPTX path, run directory, report, and editability summary.

Built for controlled automation.

No blind one-shot generation. EasyToPPT uses gates, manifests, reports, and repair loops so the output stays understandable.

1
Analyze Read source material or slide images, extract intent, page structure, text, visual regions, and constraints.
2
Plan Create approved outlines, per-slide content, region manifests, and style directions before final output.
3
Generate Use Image2-style generation for slide images, or OCR plus PPTX construction for editable decks.
4
Validate Check page order, Chinese text quality, figure preservation, editable object boundaries, and final delivery paths.

What this project says.

EasyToPPT is not another PPT template. It is a structured agent workflow for presentation generation, reconstruction, QA, and repair.

Image route:
source material -> approved outline -> style collages -> final slide PNGs

Editable route:
slide image -> OCR -> clean background -> native/hybrid/assets -> editable PPTX

Principle:
preserve truth, preserve visual fidelity, expose the editability boundary.
